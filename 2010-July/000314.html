Marcus Schäfer writes:
 > Hi,
 > 
 > > my Name is Sascha Manns from open-slx. I had heard that the
 > > development of sax2 is closed. Is that right?
 > 
 > This is correct yes. We are maintaining old versions as long
 > as the maintenance period lasts but there is no active new
 > development for sax2 anymore.
 > 
 > The main reason for this is because Xorg doesn't require
 > a static configuration file anymore. SaX2's main task was to
 > create that file and because of the fact that this base
 > requirement doesn't exist anymore the project is at the
 > end of its lifecycle

Today everything focuesses on autodetection and on-the-fly 
configuration during run time. This is considered the future
by many in the upstream community.

X can still take a static configuration - and for some corner
cases there may still be a need for this - however the format
of the configuration has changed: for example instead of a
single config file X will take each section now in a separate
file. This will help creating only partial configurations.
Still the amount of changes needed for SaX2 would require an
effort that doesn't seem to be warrented for the corner cases 
which may still exist.
Still, if someone's interested in picking it up nothing should
stop this person.
See also the announcement I've made last December:
http://lists.opensuse.org/opensuse-factory/2009-12/msg00017.html
