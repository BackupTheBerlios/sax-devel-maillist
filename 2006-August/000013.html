> Hold on, I checked again - this is dying differently. In wlan.c actually - 
> well I do have wpa and wlan support on my system, so I am not sure WHY that 
> dies, but it does bring up the interesting question of why an X tool needs to 
> handle wifi cards ? Can you suggest a work-around ?

The problem is that sax is using a hardware library named hwinfo.
This library is able to detect all kind of hardware which includes
wlan cards as well. The source for this library is open source but
the Author didn't had the time to establish an open connection for
the source. According to this reason there is a copy of the source
maintained in the sax2 code as long as the hwinfo source code
is not public accessable.


> wlan.c: In function `wpa_driver_wext_set_wpa':
> wlan.c:1065: error: `SIOCSIWAUTH' undeclared (first use in this function)

These defines are part of:

	/usr/include/linux/wireless.h

I assume your glibc-devel library headers are too old to be able
to build the wpa support. sax2 doesn't make use of this code so
a very nasty workaround could be to define the missing symbols
