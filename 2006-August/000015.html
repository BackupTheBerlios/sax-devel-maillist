the problem is as long as you didn't install it into your system
sax2 cannot work. The major reason for this are the perl modules.
I'm using MakeMaker to create the Makefiles from the Makefile.PL
input source. The modules are autoloaded and therefore it is needed
that perl installs the modules into the perl distribution. With
my system the location for e.g FBSet.pm is

	/usr/lib/perl5/vendor_perl/5.8.8/i586-linux-thread-multi/FBSet.pm

I you don't mind install the build or even better you can create a
package set :-) This would be really great. Another solution is to
add -- use lib "<path-to-local-build>"; -- whenever needed but this requires
some places within sax to be changed. I think this is not an option.
